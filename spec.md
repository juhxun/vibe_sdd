# 오늘의 할 일 앱 - 기능 명세서

## 개요
사용자가 일상의 할 일을 관리할 수 있는 웹 애플리케이션입니다. 할 일 추가, 완료 처리, 삭제, 필터링 기능을 제공하며, 모든 데이터는 서버 재시작 후에도 유지됩니다.

---

## 핵심 기능

### S01. 할 일 추가
**사용자 스토리**: 사용자는 새 할 일을 입력해 목록에 추가할 수 있다.

**요구사항**:
- 입력 필드에 텍스트를 입력하고 엔터 또는 추가 버튼을 누르면 목록에 추가된다.
- 빈 값이거나 공백만 있는 입력은 추가되지 않는다.
- 빈 입력 시도 시 사용자에게 안내 메시지를 표시한다. 예: "할 일을 입력해주세요."
- 추가 완료 후 입력 필드는 초기화된다.
- 새 항목은 목록의 맨 아래에 추가된다.

**완료 조건**:
- pytest에서 `test_create_todo_success` 통과
- pytest에서 `test_create_todo_empty_fails` 통과

---

### S02. 할 일 완료 토글
**사용자 스토리**: 사용자는 할 일을 완료 처리할 수 있고, 완료된 항목은 취소선과 흐린 색으로 표시된다. 완료된 항목을 다시 미완료 상태로 되돌릴 수 있다.

**요구사항**:
- 각 할 일 항목에 체크박스가 있다.
- 체크박스를 클릭하면 `completed` 상태가 토글된다.
- 완료 상태 항목은 취소선(`text-decoration: line-through`)과 회색 색상으로 표시된다.
- 미완료 상태로 다시 변경할 수 있다.
- 토글 즉시 화면이 업데이트되고, 남은 할 일 개수도 갱신된다.

**완료 조건**:
- pytest에서 `test_toggle_todo_completion_success` 통과
- pytest에서 `test_toggle_todo_not_found` 통과

---

### S03. 할 일 삭제
**사용자 스토리**: 사용자는 할 일을 삭제할 수 있다. 실수 방지를 위해 삭제 전 확인 절차가 있다.

**요구사항**:
- 각 할 일 항목에 삭제 버튼(X 또는 휴지통 아이콘)이 있다.
- 삭제 버튼을 클릭하면 확인 대화상자(`confirm()`)를 띄운다. 예: "이 항목을 삭제하시겠습니까?"
- 사용자가 확인하면 항목이 삭제되고, 화면이 즉시 갱신된다.
- 취소하면 아무 일도 일어나지 않는다.
- 남은 할 일 개수도 즉시 갱신된다.

**완료 조건**:
- pytest에서 `test_delete_todo_success` 통과
- pytest에서 `test_delete_todo_not_found` 통과

---

### S04. 남은 할 일 개수 표시
**사용자 스토리**: 화면 상단에 "남은 할 일 N개"가 항상 표시되고, 추가/완료/삭제 시 즉시 갱신된다.

**요구사항**:
- 남은 할 일 개수는 `completed = false`인 항목의 개수이다.
- 화면 상단에 항상 표시된다. 예: "남은 할 일 3개"
- 할 일 추가/완료 토글/삭제 시 즉시 갱신된다.
- 할 일이 없을 때는 "남은 할 일 0개"로 표시된다.

**완료 조건**:
- pytest에서 `test_count_active_todos` 통과
- 화면 렌더링 테스트에서 카운트 업데이트 확인

---

### S05. 필터 기능
**사용자 스토리**: 사용자는 전체 / 진행중 / 완료 3가지 필터로 목록을 걸러 볼 수 있다.

**요구사항**:
- 화면에 3개의 필터 버튼이 있다: "전체", "진행중", "완료"
- "전체": 모든 할 일 항목을 표시한다. (기본값)
- "진행중": `completed = false`인 항목만 표시한다.
- "완료": `completed = true`인 항목만 표시한다.
- 필터 적용 시 화면이 즉시 갱신되고, 활성 필터 버튼이 강조된다.
- 남은 할 일 개수는 항상 전체 미완료 개수를 표시한다. (필터 적용과 무관)

**완료 조건**:
- pytest에서 `test_filter_active_todos` 통과
- pytest에서 `test_filter_completed_todos` 통과
- pytest에서 `test_filter_all_todos` 통과

## 품질 요구사항

### Q01. 데이터 영속성
**요구사항**:
- 브라우저를 새로고침하거나 서버를 재시작해도 할 일 목록이 유지된다.
- 모든 데이터는 SQLite 데이터베이스에 저장된다.
- 데이터베이스 파일 경로는 `.env` 파일의 `DATABASE_URL` 환경변수로 관리된다.

**완료 조건**:
- 통합 테스트에서 서버 재시작 후 데이터 유지 확인

---

### Q02. 반응형 화면
**요구사항**:
- 모바일 화면(360px 폭)에서도 사용할 수 있는 반응형 설계이다.
- 데스크톱(1024px+), 태블릿(768px+), 모바일(360px+) 모두에 최적화된다.
- 텍스트, 버튼, 입력 필드는 터치/클릭하기 편한 크기(최소 44x44px)이다.
- 화면 너비에 따라 레이아웃이 자동으로 조정된다.

**완료 조건**:
- 360px 폭에서 모든 요소가 스크롤 없이 보이고 조작 가능한지 수동 확인
- 개발자 도구의 반응형 디자인 모드에서 테스트

---

### Q03. 빈 상태 안내
**요구사항**:
- 할 일이 하나도 없을 때는 친절한 빈 상태(empty state) 안내 문구를 보여준다.
- 예: "아직 할 일이 없습니다. 새로운 할 일을 추가해보세요!"
- 빈 상태 메시지는 적절한 위치에 표시되고, 사용자가 행동을 취하도록 유도한다.

**완료 조건**:
- 모든 할 일을 삭제했을 때 빈 상태 메시지 표시 확인
- 필터로 진행중/완료를 선택했을 때도 해당 상태가 비어있으면 메시지 표시

---

## 비기능 요구사항

- **성능**: 페이지 로드 시간 < 2초, API 응답 시간 < 200ms
- **브라우저 호환성**: 최신 크롬, 파이어폭스, 사파리, 엣지 지원
- **보안**: SQL 인젝션 방지(ORM 사용), CORS는 필요시만 구성
- **확장성**: 향후 데이터 스토어 변경 용이하도록 DB 추상화

---

## 제약사항

- 사용자 인증 기능은 포함하지 않는다.
- 협업/공유 기능은 포함하지 않는다.
- 클라우드 동기화는 포함하지 않는다.
- 다국어 지원은 포함하지 않는다(한국어만 지원).
